# ループ定義: [100マス計算] Issue実装ループ

## 概要

sansu-100（100マス計算）に関連するGitHub issueを順番に実装していくループ。

## ループ設定

- **間隔**: 25分ごと
- **Cronパターン**: `*/25 * * * *`
- **モデル方針**:
  - 基本: `claude-sonnet-4-6`（トークン節約）
  - 難しいタスク: `claude-opus-4` 高effort
  - `/ai-mix` スキルに従う

## 実行プロンプト

```
[100マス計算]に関連するissueを実装していって下さい。基本的にトークンは無駄遣いしないように、sonnetを使うようにして下さい。ただし、少し難しいタスクの時はopus高で対応して下さい。あとは/ai-mix に従って下さい。
```

## Issueの優先順位方針

1. `gh issue list --state open` でopen issueを確認
2. `[100マス計算]` ラベルのissueを優先
3. 実装規模が小さい（ロジック変更）ものから大きい（新機能追加）順に着手
4. 1ループで1issueを完結させるのが理想。大きい場合は分割して進める

## 実装後の手順

1. `npm run build` でビルド確認（フロント・API両方）
2. ユニットテスト: `NVM_DIR="$HOME/.nvm" && [ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh" && nvm use 22 && npm test`
3. **動作確認は Playwright E2E で実施**（UIの挙動・表示まで確認）
   - テストは `tests/sansu/` に `.spec.ts` で追加する
   - 実行: `npx playwright test --config playwright.config.sansu.ts`（SWA環境が必要: `npm run sansu:dev`）
   - 手動起動済みなら `reuseExistingServer: true` で自動再利用される
4. PRを作成して `gh pr create`

このループを再開するには:
```
/loop 25分ごと [100マス計算]に関連するissueを実装していって下さい。基本的にトークンは無駄遣いしないように、sonnetを使うようにして下さい。ただし、少し難しいタスクの時はopus高で対応して下さい。あとは/ai-mix に従って下さい。今後もこのissue実装を依頼することがあると思うのでloopの定義をまずは作って保管しておいて下さい。
```
